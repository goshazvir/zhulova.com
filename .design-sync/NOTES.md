# design-sync notes — zhulova

Repo-specific gotchas for syncing the Zhulova Design System to claude.ai/design.

## Repo shape

- This is an **Astro site**, not a standalone component package. The design
  system lives in `src/design-system/` with a barrel at
  `src/design-system/index.ts`.
- **No dist build.** The converter bundles the barrel directly with esbuild:
  `--entry ./src/design-system/index.ts`. There is no `dist/` to build first.
- **Props are hand-written** in `cfg.dtsPropsFor`. Auto-extraction yielded
  `{[key: string]: unknown}` (default exports + `ComponentProps<'el'>` intersections
  the extractor couldn't flatten), so each component's real props are pinned in
  config. Keep `dtsPropsFor` in sync when a component's props change.
- `componentSrcMap` pins all 10 components; several share one file
  (Typography→Heading/Text, Section→Section/Container, Badge→Badge/Stat).

## Styling

- Styling is **Tailwind utility classes**, not a shipped component stylesheet.
  `cfg.cssEntry` → `.design-sync/ds-styles.css`, generated by compiling Tailwind
  scoped to `src/design-system` **and** `.design-sync/previews`.
- **Regenerate CSS** (do this before `package-build` whenever component or
  preview classes change):
  ```sh
  npx tailwindcss -c .design-sync/tw.config.mjs -i .design-sync/tw-input.css -o .design-sync/ds-base.css --minify
  printf "@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Playfair+Display:wght@600;700&display=swap');\n" > .design-sync/ds-styles.css
  cat .design-sync/ds-base.css >> .design-sync/ds-styles.css && rm .design-sync/ds-base.css
  ```
- Tokens (navy/gold/sage colors, fonts) come from `tailwind.config.mjs`
  `theme.extend`. The scoped config `.design-sync/tw.config.mjs` reuses it.
- Fonts (Playfair Display, Inter) load via a remote Google Fonts `@import` →
  `[FONT_REMOTE]`, expected and non-blocking (no font files shipped).

## Preview overrides

- **Modal** uses `position: fixed`. Its preview (`previews/Modal.tsx`) wraps the
  Modal in a `transform: translateZ(0)` container so the overlay renders inside
  the card instead of escaping to the viewport. Don't remove the wrapper.
- `cfg.overrides`: Modal = single + viewport 600x520; Section/Container/Card =
  cardMode column (they're full-width).

## Known render warns

- `[FONT_REMOTE]` for "Inter" / "Playfair Display" — intentional (remote webfont).

## Re-sync risks

- **`ds-styles.css` is generated, not authored truth.** If component utility
  classes or preview classes change and the CSS is NOT regenerated (step above)
  before `package-build`, cards/designs render unstyled. Always regenerate on
  re-sync when DS source changed.
- **`dtsPropsFor` is hand-maintained** and drifts silently if a component's
  props change. Update it to match the source when props change.
- **Tokens live in `tailwind.config.mjs`** — a token change there needs a CSS
  regeneration to take effect in the synced bundle.
- All 10 previews are authored compositions in `.design-sync/previews/` and
  carry forward; they assume the current component APIs.
